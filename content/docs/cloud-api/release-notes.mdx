_Current deep learning model:_ **v2.40.0**(Jun 01, 2026)

## June 01, 2026

The Python `cochl` library and Dashboard have been released with Integrated API `v1.0.0`, deployed alongside model `v2.40.0`. Alongside the existing environment sound recognition, the Cloud API now offers:

  

*   **[Speech Analysis](/docs/cloud-api/speech-analysis)**—Transcription (speech-to-text) plus registered speaker matching, renamed and expanded from the former Speaker Recognition.
*   **[Audio Insights](/docs/cloud-api/audio-insights)**—context awareness for audio scenes and a comprehensive summary covering environment, conversation topics, and events.
*   New Integration API endpoint `POST /api/v1/analyze_file` lets you combine Sound Event Detection, Speech Analysis, and Audio Insights in a single request. See [Audio Insights](/docs/cloud-api/audio-insights#2-run-audio-insights) for the OpenAPI link.
*   Approximately **3% AP improvement** on Sound Event Detection with stronger robustness against background noise and overlapping voices.

## July 07, 2022

Two new concepts have been implemented:

*   `multi-tag`: multiple sounds can be recognized at the same time.
*   `no service`: all the sounds can be inferenced regardless of their category: no need to specify a service anymore.

**Breaking changes**

Due to multi-tag concept, the **‘Read status’** request response format has been modified.

* * *

## March 15, 2022

Two new parameters are available from client side when creating a session:

*   `service`: set service to inference different events with same project / api key
*   `metadata`: those metadata will later be available on analytics Dashboard.

**Breaking changes**

Events for each service have been refactored.
