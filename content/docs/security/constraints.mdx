Known limitations and unsupported configurations.

## Network requirement

Cochl Security requires outbound HTTPS connectivity to Cochl Cloud (`api.cochl.ai/security`) for license validation, dashboard sync, and event monitoring.

Air-gapped or fully network-isolated environments are not supported by the standard product. Reach out via the [contact form](https://www.cochl.ai/contact-us/) to discuss Enterprise options.

## Unsupported environments

| Environment | Reason |
| --- | --- |
| Air-gapped environments | License validation and dashboard sync require outbound HTTPS |
| Customer-owned inference servers | Only Cochl-certified hardware is supported for AI inference |

## On-Prem & custom deployment

Cochl Security is designed for open network (cloud-connected) environments. On-premise deployment is possible on a case-by-case basis for Enterprise customers. Reach out via the [contact form](https://www.cochl.ai/contact-us/) to discuss.
